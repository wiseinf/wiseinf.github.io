Cluster Optimizer version 1.2.0 has been released

Mainly describes the new features of the cluster optimization platform Cluster Optimizer version 1.2.0.
2024-05-20

Cluster optimization platform Cluster Optimizer version 1.2.0 has been released.

Key features include:

  1. Authentication and authorization: Supports user authentication and permissions, providing users with two roles: Administrator and User. Administrators have all operation permissions, and users only have viewing permissions.
  2. Cluster access account: Supports management of user’s cluster access account, and can set a cluster-level cluster access account for each user, including the user’s cloud service provider account, and the cluster’s Kubeconfig and other configurations. Users can operate the corresponding cluster through this identity. At the same time, if the cluster supports automatic Kubeconfig update, the user’s cluster access account can also be updated regularly when in use.
  3. Automated actions: Support application-level resource recommendation, unset resource quotas and memory overflows, and delete unused persistent volumes and other automated actions. Support standard workloads Deployment, StatefulSet, DaemonSet and CronJob. Job is not supported, and its resource quota (Resource Quota) cannot be modified.
  4. Task management: Supports daily task management operations such as creating, running, re-running, canceling, viewing task logs, and deleting tasks.

To quickly deploy and try Cluster Optimizer, see: Free Trial.

Cluster Optimizer version 1.1.0 has been released

Mainly describes the new features of the cluster optimization platform Cluster Optimizer version 1.1.0.
2024-04-09

Cluster optimization platform Cluster Optimizer version 1.1.0 has been released.

Key features include:

  1. AWS Cloud EKS integration: Supports importing AWS EKS clusters and viewing optimization suggestions for the clusters.
  2. GPU node: Supports viewing GPU nodes with low GPU utilization.
  3. Quick resource association: Supports quick association of resources with corresponding departments, groups, and users through resource tags, and can be displayed on the Web interface.
  4. Email notification: For specific optimization strategies, users can subscribe to email notifications to learn about corresponding optimization suggestions through email.
  5. Cluster status: Can display the status of Kubernetes and Metrics services imported into the cluster. For abnormal conditions, the cause of the abnormality can be quickly discovered.
  6. Support automatic creation of database Schema: You can use the parameter createDatabaseIfNotExist=true in the database connection configuration item to indicate whether to automatically create the database if the database does not exist. When the specified user name and password have permission to create a database, the system will automatically create the database.

To quickly deploy and try Cluster Optimizer, see: Free Trial.

Cluster optimization platform Cluster Optimizer version 1.0.0 has been released.

Key features include:

  1. Alibaba Cloud ACK integration: Supports importing Alibaba Cloud ACK clusters and viewing optimization suggestions for the cluster, including node groups, nodes, disks, persistent volumes, and applications.
  2. Settings: support create, edit and delete clusters and cloud provider’s account.
