Periodic notifications of optimization suggestions

Describes how to set up a periodic optimization suggestion notification mechanism through Cluster Optimizer to facilitate obtaining corresponding optimization suggestions via email.

Describes how to set up a periodic optimization suggestion notification mechanism through Cluster Optimizer to facilitate obtaining corresponding optimization suggestions via email.

Overview

During the operation of Kubernetes clusters, we often need to obtain key indicators or information regularly and communicate this information to relevant users through emails or other notification methods. The periodic notification mechanism allows users to make necessary adjustments and optimization operations based on the information received.

The cluster optimization platform can automatically monitor possible problems and potential optimization points in the cluster, such as low node utilization, application memory overflow, unused Persistent Volume, etc., and generate optimization suggestions containing detailed information and recommended parameters, and notify users via email. . For example, when an application memory overflow is detected, the system will automatically collect relevant information and send this information to the person in charge or responsible person of the application. This not only improves the efficiency of problem response, but also ensures efficient utilization of cluster resources and stable operation of applications. In this way, users can more proactively manage and maintain clusters and applications in the cluster, improving overall operational efficiency.

Usage scenarios

Users want to obtain common optimization advice information through emails, such as low node utilization, application memory overflow, Persistent Volume is not used, etc., and perform corresponding operations based on these optimization advice information. This article will take configuring node group optimization recommendations as an example to outline how to configure Cluster Optimizer so that it can provide periodic optimization recommendation notifications.

Suppose we expect to notify user Jay (email address is jay@wiseinf.com) via email at 8 a.m. every day of the node group optimization suggestions in the cluster. The specific preconditions and configuration process are as follows.

Assume that the user has the following needs:

  1. Send the node group optimization suggestion results to user Jay (the email address is jay@wiseinf.com)
  2. Receive notifications via email at 8 a.m. daily

The cluster optimization platform provides notifications to users through the following two components:

  • Notification channel: Provides basic configuration parameters for sending notifications. For example, for email notifications, typical configurations include host address, port, user name and password, etc. Notification channels can be used by multiple notification rules.
  • Notification rules: Provide configuration parameters, recipient addresses, etc. for constructing notification content.

The following describes how to configure notification channels and notification rules through the cluster optimization platform so that it can meet the above user needs.

Preconditions

  1. Cluster Optimizer has been deployed. For details, please refer to Installation and Deployment
  2. The cluster information has been imported. For details, please refer to Import cluster
  3. SMTP server related information, including host address, port, user name, password, etc.

Configuration steps

  1. Configure notification channels: Go to the notification channel management page from Settings --> Notification Channels. On the notification channel page, click the Add button to pop up the Add Notification Channel dialog box. In the notification channel dialog box, enter the corresponding information, and then click the OK button, as shown in the figure below.

    Among them, the notification channel name is set to email, the host address is set to smtp.wiseinf.com, the port is set to 465, the user name is set to optimizer@wiseinf.com, password, etc., where the SMTP server host address is , port, username, password and other information are the SMTP server related information obtained in the prerequisites and need to be filled in according to the actual situation.

    Note: For the description of the configuration items of the notification channel dialog box, please refer to [Add Channel]/docs/user-guide/management/channel/#Add Channel

  2. Configure notification rules: Go to the notification rule management page from Settings --> Notification Rules. On the notification channel page, click the Add button to pop up the Add Notification Rule dialog box. The notification rules dialog box mainly includes the following information: notification recipient information, focused optimization suggestions, notification channels, and matching rules. Please enter the corresponding information and click the OK button, as shown in the figure below.

    Among them, the receiver is set to jay@wiseinf.com, the notification channel selects the email notification channel created in Step 1, the scheduling time is set to 8 am every day, the optimization strategy selects the node group recommendation, and the cluster Just select the cluster imported in the preconditions, as shown in the figure below.

Using the recommended values ​​can effectively reduce costs while making dynamic load more effective in utilizing the ability to automatically expand and shrink to maximize the stability of collective applications.

Learn more

The cluster optimization platform provides a series of capabilities to facilitate users to easily obtain optimization recommendation notifications.

The cluster optimization platform provides a series of optimization strategies, including low utilization nodes, low utilization disks, Persistent Volume identification, application resource recommendation, memory overflow, etc. The notification rules in the cluster optimization platform provide a series of matching rules. Users can choose different matching rules according to their needs, including matching for resource owners (including departments, groups, users) and namespace matching. Users can use personalized custom templates to achieve personalized email notifications.
