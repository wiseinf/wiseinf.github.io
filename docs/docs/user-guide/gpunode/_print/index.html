GPU Node

Describes GPU node related information and optimization suggestions.

Describes GPU node related information and optimization suggestions.

View GPU node information

Click the GPU Node menu on the left menu bar of the platform to display the GPU node page, as shown in the figure below.

GPU nodes are divided into 3 areas:

  1. Cluster selection area: used to select clusters.
  2. Query Area: Used to filter GPU nodes by multiple conditions.
  3. List area: Displays GPU node information, its owner, corresponding configuration and utilization, etc.

After selecting a cluster, the GPU node list area will display a list of GPU nodes that meet the conditions in the cluster, as shown below:

The list area is divided into five columns:

The list area display is divided into 5 columns:

  1. Node information bar: Displays the GPU node name, instance ID and region information.
  2. Owner Column: Displays the department, group and specific personnel of the node owner.
  3. Settings bar: Displays the instance type, number of CPU cores, memory size, number of GPUs, and GPU type of the GPU node.
  4. Details column: Displays the information provided by the strategy. For example, for low-utilization node strategies, the average utilization of CPU and memory will be observed and compared with the thresholds set by the platform to help users adjust node instance types.
  5. Operation column: Displays the corresponding operations on the node.

Adjust nodes

Click the Adjust link in the Action Bar to open the Adjust Node dialog box, as shown in the figure below.
