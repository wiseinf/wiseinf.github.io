View cluster

Click Cluster on the left menu bar of the platform to display the cluster overview page, as shown in the figure below.

The main display area will display the list of clusters managed by the platform. Including cluster (cluster name and cluster ID), cluster region, Access Token name, cloud service provider, Prometheus address, health status and other information.

In the commercial version, additional cluster status and operation bars are provided, as shown in the following figure:

Cluster status displays the corresponding status of the cluster to users, making it easier for users to understand the cluster status. Mainly include:

  1. Running: The cluster is running normally.
  2. Sleeping: The cluster is entering sleep state.
  3. Hibernate: The cluster has hibernated. In the dormant state, the cluster has released corresponding node resources, and the cluster usage is limited at this time.
  4. Waking up: The cluster is transitioning to the running state.

In the cluster operation column, users can operate the cluster, including:

  • Set cluster hibernation *Hibernate immediately
  • Wake up immediately
  • log

Set cluster hibernation

Cluster hibernation reduces cluster resource usage and saves costs by properly managing the nodes in the cluster. Assume that the cluster starts sleeping at 9 pm from Monday to Friday and wakes up at 8 am from Monday to Friday (that is, ending hibernation). The machine cost savings can be as high as 54.8%. Setting cluster hibernation allows you to set the sleep time and wakeup time of the cluster. During the sleep time, the status of the load in the cluster is stored, and the node resources in the cluster are gradually released. During the wake-up time, the system gradually restores the node resources in the cluster and restores the original state of the load. Cluster hibernation is especially suitable for offline environments such as development and testing.

Click Set Cluster Hibernation in the operation bar, and the dialog box will pop up as shown below:

Select Enable in the Whether to enable sleep option, then set the specific sleep time and wake up time, and click the OK button. The default sleep time is 9pm from Monday to Friday, and the default wake-up time is 8am from Monday to Friday.

If you want to cancel cluster hibernation, select Disable in the Whether to enable hibernation option and click the OK button.

When the cluster is dormant, the cluster access account provided by the user will be used to operate the cluster. You need to set up a cluster access account. For details, please see Setting up a cluster access account.

Note: The interval between sleep time and wake up time must be greater than 2 hours.

Hibernate now

Move the mouse to ... and click the Hibernate Now button to hibernate the cluster immediately.

Wake up immediately

Move the mouse to ... and click the Wake Up Now button to wake up the cluster immediately.

log

Move the mouse to ... and click the Log button to view logs related to cluster sleep or wake-up.
