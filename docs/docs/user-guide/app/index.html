Application / Workload

Describe application details and optimization suggestions.
Tags:
  • Cluster Optimizer
  • Application Optimizer
Categories:
  • Cluster Optimization

Application details and optimization suggestions are described.

View application

Click the Application menu on the left menu bar of the platform to display the application page, as shown in the figure below.

The application page is divided into 3 areas:

  1. Cluster selection area: Select the corresponding cluster.
  2. Query area: You can query by namespace, load name, load type, container and policy name. Currently, the platform supports resource recommendation, unset resource quota, and memory overflow strategies.
  3. List area: Display name, container, current settings, policy name, details and operation bar. Different policy names will have some differences in the displayed information.

After selecting a cluster in the cluster selection area, the platform aggregates the results of all strategies by default and displays them in a list, as shown in the figure below.

Resource recommendation

View resources and recommend related applications

If the policy name is Resource Recommendation, the list area will be as shown in the figure above. The list area display is divided into 6 columns, which are:

  1. Name column: Displays the payload name, namespace and payload type.
  2. Container bar: Displays the container name.
  3. Owner column: Displays the department, group and specific personnel of the node owner.
  4. Current Settings: Displays the CPU Request, CPU Limit, Memory Request and Memory Limit of the container.
  5. Details: Display average CPU usage, maximum CPU usage, average memory usage, maximum memory usage, CPU Request recommended value, CPU Limit recommended value, Memory Request recommended value, Memory Limit recommended value, strategy Name and last updated.
  6. Operation column: Displays the corresponding operations on the application.

Adjust resource quota

Click the Adjust link to open the Adjust Resource Quota dialog box, which displays application details, recommended values ​​and corresponding adjustment statements.

Resource quota is not set

View applications that do not have resource quotas set

If the policy name is No resource quota set, the list area is displayed as shown below:

The list area display is divided into 6 columns, which are:

  1. Name column: Displays the payload name, namespace and payload type.
  2. Container bar: Displays the container name.
  3. Owner Column: Displays the department, group and specific personnel of the node owner.
  4. Current Settings: Shows whether the container’s CPU Request, CPU Limit, Memory Request and Memory Limit have been set.
  5. Details: Displays the recommended value of CPU Request, recommended value of CPU Limit, recommended value of Memory Request, recommended value of Memory Limit, policy name and latest update time.
  6. Operation Bar: Displays the corresponding operations on the application.

Memory overflow

View memory overflow applications

If the policy name is Memory Overflow, the list area will be displayed as shown below:

The list area display is divided into 6 columns, which are:

  1. Name column: Displays the payload name, namespace and payload type.
  2. Container bar: Displays the container name.
  3. Owner Column: Displays the department, group and specific personnel of the node owner.
  4. Current settings: Displays the memory request and memory limit of the container.
  5. Details: Displays the latest OOM time, recent OOM Pod, number of OOMs, and latest update time.
  6. Operation column: Displays the corresponding operations on the application.
