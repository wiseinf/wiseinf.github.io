Review applications / workloads

Click on the Application menu on the left menu bar of the platform to reveal the application page, as shown below.

The application page is divided into three areas:

  1. Cluster Selection Area: Select the appropriate cluster.
  2. Query Area: Query by namespace, load name, load type, container, and policy name. The platform currently supports resource recommendation, resource quotas unset, and out of memory strategies.
  3. List Area: Displays the name, container, current settings, policy name, details, and an operation bar. The displayed information may vary depending on the strategy name.

After selecting a cluster in the Cluster Selection Area, the platform aggregates and displays the results of all strategies by default in a list, as shown in the figure below.

Resource recommendation

Review resource recommendations

When the strategy name is Resource Recommendation, the list area is displayed as shown in the figure above. The list area is divided into six columns:

  1. Name: Displays the application name, namespace, and application type.
  2. Container: Shows the container name.
  3. Owner: Indicates the department, group, and specific personnel of the node owner.
  4. Current Settings: Shows the CPU Request, CPU Limit, Memory Request, and Memory Limit for the container.
  5. Details: Includes average CPU usage, maximum CPU usage, average memory usage, maximum memory usage, recommended values for CPU Request, CPU Limit, Memory Request, and Memory Limit, strategy name, and last updated time.
  6. Actions: Displays the available operations for the application.

Resource quota unset

Review resource quota unset

When the strategy name is Resource Quota Unset, the list area appears as shown below:

The list area is divided into six columns:

  1. Name: Displays the application name, namespace, and application type.
  2. Container: Shows the container name.
  3. Owner: Displays the department, group, and specific personnel of the node owner.
  4. Current Settings: Indicates whether the container’s CPU Request, CPU Limit, Memory Request, and Memory Limit have been set.
  5. Details: Shows the recommended values for CPU Request, CPU Limit, Memory Request, Memory Limit, policy name, and last update time.
  6. Operations: Displays the available actions for the application.

Out of memory

View memory overflow applications

When the strategy name is Out of Memory, the list area appears as shown below:

The list area is divided into six columns:

  1. Name: Displays the application name, namespace, and application type.
  2. Container: Shows the container name.
  3. Owner: Displays the department, group, and specific personnel of the node owner.
  4. Current Settings: Displays the memory request and memory limit of the container.
  5. Details: Shows the latest OOM time, recent OOM Pod, number of OOMs, and last update time.
  6. Operations: Displays the available actions for the application.
