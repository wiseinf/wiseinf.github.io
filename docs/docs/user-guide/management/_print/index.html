1 - User Management

Describe related operations of user management.

Describe related operations of user management.

Preconditions

*Managing users requires administrator rights. *Available only in commercial version.

View user list

Enter the user management page from Settings --> Users, as shown below.

The user management page will display notification channels in the form of a list. It mainly includes 5 columns:

  • Username: Display user name.
  • Full name: Display user name.
  • Email address: Displays the user’s email address.
  • Phone number: Displays the user’s phone number.
  • Role: Display user role.
  • Operation: Operation bar, including editing, deletion and other operations.

Add user

Click the Add button at the top of the user management page, and the Add User dialog box will pop up, as shown below.

Please see the table below for specific field names and descriptions.

Field name Description
Name User name, required
Last name User last name, optional
First Name Username, optional
Email User email, required
Password User login password, required
Confirm password Confirm user login password, required
Mobile phone number User mobile phone number, required
User role Currently, the system provides two roles: administrator and user. The default is user, which is required

Enter the corresponding information according to the field name and click the OK button to add the user.

Update user

Click the Edit button in the list of user management pages, and the edit user dialog box will pop up, as shown below.

The specific field names and descriptions are the same as [Add User] (#Add User) and will not be repeated.

Delete user

Click the Delete button on the user management page to delete the user.

Before deleting the user, a deletion confirmation dialog box will pop up. After confirmation, the deletion operation will be performed.

2 - Notification Channel

Describes how to set up notification channels.

Describes how to manage notification channels, including viewing, adding, updating, or deleting notification channels.

View channels

Enter the notification channel management page from Settings --> Notification Channels, as shown below.

The notification channel management page will display notification channels in the form of a list. It mainly includes 5 columns:

  • Name: Notification channel name.
  • Channel type: Notification channel type.
  • Connection parameters: Displays the connection parameters of this channel type.
  • Other parameters: display other parameters
  • Operation: Operation bar, including editing, deletion and other operations.

Add channel

Click the Add button at the top of the notification channel management page, select the notification channel type, and fill in the notification channel information, as shown below.

Please see the table below for specific field names and descriptions.

Field name Description
Name Notification channel name
Channel type Notification channel type, currently only supports Email
Host Email SMTP hostname, for example smtp.wiseinf.com
Port Email SMTP port, such as 25, 465
Enable SSL Whether to enable SSL, the default is not enabled
Email Email address, such as admin@wiseinf.com
Sender name Sender name, e.g. Wiseinf
Password Email SMTP Password
Parameter list User-defined parameter list in the form of key-value pairs. This parameter list will be passed to the template of the notification channel and can be referenced in the template through {{.param.parameter name}}.

Enter the corresponding information according to the field name and click the OK button to add the notification channel.

For specific values ​​of the above fields, you can consult your email service provider or email system administrator.

Update channel

To update notification channels, click the Edit button in the list of notification channel management pages to update the notification channel information, as shown below.

The specific field names and descriptions are the same as [Add Channel] (#Add Channel) and will not be repeated.

Delete channel

Click the Delete button in the list of notification channel management pages to delete the notification channel, as shown below.

Before deleting the notification channel, a deletion confirmation dialog box will pop up, and the deletion operation will be performed after the user confirms.

Note: When the notification channel is used by notification rules, it will prompt that the deletion was unsuccessful. You should first delete the notification rules that use the notification channel, and then delete the notification channel.

3 - Notification Rules

Describes the details of the management notification rule.

Describes the details of the management notification rule.

View notification rules

Enter the notification rule management page from Settings --> Notification Rules, as shown below.

The notification rule management page will display notification channels in the form of a list. It mainly includes 5 columns:

  • Name: Notification channel name.
  • Channel type: Notification channel type.
  • Connection parameters: Displays the connection parameters of this channel type.
  • Other parameters: display other parameters
  • Operation: Operation bar, including editing, deletion and other operations.

Add notification rules

Click the Add button at the top of the notification rule management page, and the Add Notification Rule dialog box will pop up, as shown below.

Please see the table below for specific field names and descriptions.

Field name Description
Name Notification channel name
Receiver Notification recipient address, such as admin@wiseinf.com
Notification channel You can select the specific notification channel name through the drop-down box
Scheduling time Set the specific scheduling time of this rule.
Optimization strategy Select a specific optimization strategy
Cluster Select the cluster to focus on
Namespace Enter a namespace name, an empty value will match all namespaces
Department Enter a department name, an empty value will match all department names
Group Enter a group name, an empty value will match all group names
User Enter a username, an empty value will match all usernames
Template ID Template ID name, the default value is default
Parameter list User-defined parameter list in the form of key-value pairs. This parameter list will be passed to the template of the notification channel and can be referenced in the template through {{.param.parameter name}}. Parameters with the same name in the notification channel parameter list will be overridden.

Enter the corresponding information according to the field name and click the OK button to add notification rules.

Update notification rules

To update the notification channel, click the Edit button in the list of notification channel management pages, and the Add Notification Rule dialog box will pop up, as shown below.

The specific field names and descriptions are the same as [Add Notification Rules] (#Add Notification Rules) and will not be repeated.

Delete notification rules

Click the Delete button on the notification channel management page to delete the notification rules.

Before deleting the notification rule, a deletion confirmation dialog box will pop up, and the deletion operation will be performed after the user confirms.
