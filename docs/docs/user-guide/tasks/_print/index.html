View tasks

Click the task icon in the menu bar on the right header of the platform, as shown in the figure below.

The task management page is displayed, as shown in the figure below.

The task management page is divided into 2 areas:

  1. Query Area: Task items can be filtered based on policy name, task status and creator.
  2. List area: Displays information such as task names, policy names, configuration items, etc.

In the query area, the policy name currently provides four items: unused PV, application resource recommendation, unused resource quota, and memory overflow. Task status includes running, successful and failed.

The list area display is divided into 6 columns:

  1. Task name column: Displays the task name.
  2. Strategy name column: Displays the name of the strategy.
  3. Task parameter column: Displays the task parameters.
  4. Status column: Displays task status.
  5. Creator column: Displays the creator of the task.
  6. Operation column: Displays operations on tasks, including re-running, canceling execution, deleting, viewing logs, etc.

The creation of tasks is responsible for the corresponding policy, see:

  • Unused PV
  • Application resource recommendation
  • Unused resource quota
  • Memory overflow
