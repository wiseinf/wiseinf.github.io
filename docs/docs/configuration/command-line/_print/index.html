Command line parameters

Describes the command line parameters of the cloud native cluster optimization platform.

Describes the command line parameters of the cloud native cluster optimization platform.

Command line parameters

You can get the usage of Cluster Optimizer through --help, as follows:

$optimizer --help
Usage:
  optimizer optimizer [flags]

Flags:
      --features stringArray Enable features, available features are [APIServer, Optimizer]
  -h, --help help for optimizer

Global Flags:
      --config string config file (default is $OPTIMIZER_HOME/conf/cluster-optimizer.yaml)
      --log-enable-color Enable color in log
      --log-enable-console Enable log to console
      --log-format string Set the log format (json or text) (default "json")
      --log-level string Set the log level (debug, info, warn, error, fatal) (default "info")

The parameters are described in detail below.

Parameter description

--help

The parameter --help provides command line usage help.

-f, --features

The parameters -f, --features indicate the features started by the cluster optimization platform, where APIServer means only providing API services, and Optimizer means starting the optimizer features.

The default value is APIServer,Optimizer, which means that the automatic expansion and contraction features and the virtual capacity feature are enabled at the same time.

--config

The parameter --config indicates the configuration file path used by the cluster optimization platform. For its configuration, please refer to Configuration File

--log-enable-color

The parameter --log-enable-color indicates whether to enable the color attribute in the log. In the console interface, enabling this feature can improve readability. The default value is false.

--log-enable-console

The parameter --log-enable-console indicates whether to enable the console log. The console log can be easily accessed through the kubectl log command. The default value is false.

--log-format

The parameter --log-format is used to set the log format. The available log format is json or text. The default value is json.

--log-level

The parameter --log-level is used to set the log level. Optional values ​​include debug, info, warn, error, fatal. The default value is info.
