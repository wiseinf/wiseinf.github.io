Install the commercial version

Describes detailed steps for installing the commercial version.

This page describes detailed steps for installing the commercial version.

Preconditions

  • The community version of the cluster optimization platform has been deployed. For specific installation steps, please refer to [Installation via Kubectl] (/docs/setup/setup-using-kubectl/) and [Installation via Helm] (/docs/setup/setup-using-helm/).
  • To obtain commercial version authorization, you can obtain authorization through the company’s official website or email [Contact Us] (mailto: business@wiseinf.com).

Configure user database

  1. Open the command line console and clone the installation script to the local computer through the following command.

        git clone https://github.com/wiseinf/setup.git

    1. If you use the Gitee warehouse address, you need to replace https://github.com/wiseinf/setup.git in the above command with https://gitee.com/wiseinf/setup.git.

    2. The subsequent steps assume that the setup library has been cloned to the user’s home directory. If it has not been cloned to the user’s home directory, the file path in the command needs to be adjusted according to the actual situation.

  2. Adjust the database information in the Cluster Optimizer configuration file. Edit ConfigMap optimizer-conf in the file ~/setup/cluster-optimizer/yaml/optimizer.yaml and add the following configuration items.

    user:
  dataSource:
    name: "root:optimizer@tcp(optimizer-mysql:3306)/user?createDatabaseIfNotExist=true&charset=utf8mb4&parseTime=true&loc=Local"
    maxOpenConns: 10

Configure commercial license

  1. Adjust the database information in the Cluster Optimizer configuration file. Edit ConfigMapoptimizer-conf in the file ~/setup/cluster-optimizer/yaml/optimizer.yaml and add the following configuration items.

    license:
  certificate: {obtained authorization certificate}
  signature: {obtained authorized signature}

  2. Deploy Deployment optimizer through the following command:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/optimizer.yaml

    Note:

    1. If the Pod of Deployment optimizer has not been restarted, please use kubectl delete pod -l app=optimizer -n wiseinf-system to delete the Pod of Deployment optimizer and then re-create it. `
    2. If you want to use a deployed database, please refer to [Use Deployed Database](#Use Deployed Database).

  3. Check the log output and confirm that the Pod status of Deployment optimizer is Running.

System initialization

  1. Open the browser and visit the address http://localhost:8000/init to access the initialization page of the commercial version. Enter the user name, password, email, mobile phone number and other information on the page, and click the OK button to complete the system initialization.

  2. Visit the address http://localhost:8000/user/login, enter the user name and password to log in to the system.

  3. At this point, you have upgraded the system to the commercial version.

Use deployed database

If you want to use the deployed user database and use a new username and password to access Cluster Optimizer, please refer to the following steps:

  1. Execute the mysql command on the command line console to log in to the Mysql database. 127.0.0.1 is the Mysql database host address. You need to replace it with the actual address you use.

        mysql -h127.0.0.1 -uroot

    Note: If you use a remote database, make sure the database is accessible from the Kubernetes cluster.

  2. Create database schema user (user library).

        mysql> source ~/setup/cluster-optimizer/sql/create_user_database.sql

    Note: Please be aware of the risk of data loss. If the user library already exists in the mysql database, the above script will first delete the library and then create a new library.

  3. Create a new username and password, and set corresponding permissions for them. The username in the script is optimizer and the password is Optimizer~. You can adjust it according to your needs.

        mysql> source ~/setup/cluster-optimizer/sql/create_user_user.sql

  4. Adjust the database information in the Cluster Optimizer configuration file. Edit the ConfigMapoptimizer-conf in the file ~/setup/cluster-optimizer/yaml/optimizer.yaml and change the user name and user name in the dataSource.name value under the configuration item user The password, database address and port are adjusted to the corresponding information of the Mysql database.

  5. Restart the application and check the log output of Cluster Optimizer (as shown below) to see whether the database configuration information is consistent with your expectations.

    {
    ...
    "user": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optconf?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    ...
}

    At the same time, also check whether there are any database-related errors. If there are no related errors, it proves that a new database has been used.
