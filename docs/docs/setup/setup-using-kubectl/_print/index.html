Install via kubectl

Describes the detailed steps to install the cloud-native cluster optimization platform (Cluster Optimizer) through kubectl.

This page describes the detailed steps to install the one-stop cloud native cluster optimization platform (Cluster Optimizer).

Preconditions

  • Mysql database, version 5.7 and above
  • Kubernetes cluster, version 1.22 and above
  • The certificate of the Kubernetes cluster can create the wiseinf-system namespace and create Deployment, ConfigMap, Service and other resources under this namespace

Clone installation script

  1. Open the command line console and clone the installation script to the local computer through the following command.

        git clone https://github.com/wiseinf/setup.git

    1. If you use the Gitee warehouse address, you need to replace https://github.com/wiseinf/setup.git in the above command with https://gitee.com/wiseinf/setup.git.

    2. The subsequent steps assume that the setup library has been cloned to the user’s home directory. If it has not been cloned to the user’s home directory, the file path in the command needs to be adjusted according to the actual situation.

Deploy application

  1. Create the namespace wiseinf-system through the following command:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/namespace.yaml

  2. Create the database optimizer-mysql through the following command, its service name is optimizer-mysql, and can be accessed through the user root and the password Optimizer~:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/mysql.yaml

    If you need to bind the database to the specified node, please edit the ~/setup/cluster-optimizer/yaml/mysql.yaml file, remove the comments in the affinity field, and modify the value of the value field to Specify the node name as shown below.

      ...
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: kubernetes.io/hostname
            operator: In
            values:
            - ip-10-16-121-100.us-west-1.compute.internal
  ...

  3. Create Deployment optimizer through the following command:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/optimizer.yaml

    Note: If you want to use a deployed database, please see [Use Deployed Database](#Use Deployed Database).

  4. Check whether optimizer is running normally through the following command:

        kubectl get pods -n wiseinf-system

    If it runs normally, you can get output similar to the following, in which the currently running Pod Name is optimizer-5b956655df-gspq6:

        NAME READY STATUS RESTARTS AGE
    optimizer-5b956655df-gspq6 2/2 Running 0 1m

  5. Create port-forward to access the optimizer service through the following command:

        kubectl port-forward svc/optimizer 8000:80 -n wiseinf-system

  6. Open the browser and visit the address http://localhost:8000/.

  7. If you can see the Cluster Optimizer homepage, the application Cluster Optimizer is deployed successfully.

Use deployed database

If you want to use the deployed database and use a new username and password to access Cluster Optimizer, please refer to the following steps:

  1. Execute the mysql command on the command line console to log in to the Mysql database. 127.0.0.1 is the Mysql database host address. You need to replace it with the actual address you use.

        mysql -h127.0.0.1 -uroot

    Note: If you use a remote database, make sure the database is accessible from the Kubernetes cluster.

  2. Create database schema optconf (optimization configuration library) and optdata (optimization database).

        mysql> source ~/setup/cluster-optimizer/sql/create_database.sql

    Note: Please be aware of the risk of data loss. If the optconf library and optdata library already exist in the mysql database, the above script will first delete the library and then create a new library.

  3. Create a new username and password, and set corresponding permissions for them. The username in the script is optimizer and the password is Optimizer~. You can adjust it according to your needs.

        mysql> source ~/setup/cluster-optimizer/sql/create_user.sql

  4. Adjust the database information in the Cluster Optimizer configuration file. Edit the ConfigMap optimizer-conf in the file ~/setup/cluster-optimizer/yaml/optimizer.yaml, and change the dataSource.name value under the configuration items optdata and optconf The user name, user password, database address and port are adjusted to the corresponding information of the Mysql database.

  5. Restart the application and check the log output of Cluster Optimizer (as shown below) to see whether the database configuration information is consistent with your expectations.

    {
    ...
    "optconf": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optconf?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    "optdata": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optdata?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    ...
}

    At the same time, also check whether there are any database-related errors. If there are no related errors, it proves that a new database has been used.
