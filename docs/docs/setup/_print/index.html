1 - Setup using kubectl

Instructions for installing the Cluster Optimizer on Kubernetes.

Instructions for installing the Cluster Optimizer on Kubernetes.

Preconditions

  • Mysql database, version 5.7 and above
  • kubernetes cluster, version 1.22 and above
  • Git

Clone installation script

We assume the installation script library has been cloned into setup subdirectory of user’s home directory.

  1. Clone installation script into local using git: 
        cd ~
    git clone https://github.com/wiseinf/setup.git
  2. Navigate to the setup directory. 
       cd setup
    The installation directory contains:
    • Kubernetes YAML files in cluster-optimizer/yaml
    • SQL script in cluster-optimizer/sql/

Setup deployment Cluster Optimizer

  1. Create namespace wiseinf-system using:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/namespace.yaml

  2. Create database optimizer-mysql using:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/mysql.yaml

    Its service name is optimizer-mysql, and can be accessed by user root with password Optimizer~.

    To bind the database to a specific node, please edit the ~/setup/cluster-optimizer/yaml/mysql.yaml file, uncomment the affinity field, and modify the value field to the desired node name, as shown below.

      ...
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: kubernetes.io/hostname
            operator: In
            values:
            - ip-10-16-121-100.us-west-1.compute.internal
  ...

  3. Create deployment optimizer using:

        kubectl apply -f ~/setup/cluster-optimizer/yaml/optimizer.yaml

    Note: If you intend to use a deployed database, please refer to Using a Deployed Database.

  4. Check if optimizer is running:

        kubectl get pods -n wiseinf-system

    You can get:

        NAME                         READY   STATUS    RESTARTS   AGE
    optimizer-5b956655df-gspq6   2/2     Running   0          1m

  5. Create a port-forward to access optimizer service:

        kubectl port-forward svc/optimizer 8000:80 -n wiseinf-system

  6. Open your browser and visit url http://localhost:8000/.

  7. If you can see the home page of Cluster Optimizer, you have successfully setup optimizer.

Using a Deployed Database

If you want to use a deployed database or want to deploy the database by yourself, please follow the instructions below:

  1. Login into mysql database using:

        mysql -h127.0.0.1 -uroot

    The 127.0.0.1 is the host address of the MySQL database, which you need to replace with the actual address you are using.

    Note: If you are using a remote database, please make sure the database is accessible from the kubernetes cluster.

  2. Create database schema optconf(used to store configurations) and optdata(used to store data)

        mysql> source ~/setup/cluster-optimizer/sql/create_database.sql

    Note: Please be aware of the risk of data loss. The script will drop schema optconf and optdata first and then create new schemas. If you already have an optconf or an optdata schema, your data in this schema will be lost.

  3. Create a new user and grant permission to the new user. The user name is optimizer, password is Optimizer~, you can change it to your own.

        mysql> source ~/setup/cluster-optimizer/sql/create_user.sql

  4. Adjust the database information in the Cluster Optimizer configuration file. Edit the ConfigMap optimizer-conf in the file ~/setup/cluster-optimizer/yaml/optimizer.yaml, and adjust the values of dataSource.name under the configuration items optdata and optconf to match the corresponding information of the MySQL database, including the username, password, database address, and port.

  5. Restart the application and check the log output of the Cluster Optimizer to verify if the database configuration information matches your expectations. For example:

    {
    ...
    "optconf": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optconf?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    "optdata": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optdata?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    ...
}

    At the same time, also check for any database-related errors. If there are no relevant errors, it indicates that the new database has been successfully used.

2 - Setup using Helm

Instructions for installing the Cluster Optimizer on Kubernetes using Helm.

Instructions for installing the Cluster Optimizer on Kubernetes using Helm.

Preconditions

  • Mysql database, version 5.7 and above
  • Kubernetes cluster, version 1.22 and above
  • The certificate of the Kubernetes cluster enables the creation of the wiseinf-system namespace and the creation of resources such as Deployment, ConfigMap, Service, etc. under this namespace.
  • Helm, version 3.0.0 and above

Deploy Cluster Optimizer

  1. To add the Helm repository cluster-optimizer, use the following command:

        helm repo add cluster-optimizer https://wiseinf.github.io/cluster-optimizer/

  2. 通过如下命令更新仓库：To update the repository, use the following command:

        helm repo update

  3. To view the charts contained in the repository cluster-optimizer, use the following command: 通过如下命令查看仓库cluster-optimizer包含的Charts：

        helm search repo cluster-optimizer

    The output is as follows: 输出如下：

    NAME                                    CHART VERSION   APP VERSION     DESCRIPTION                       
cluster-optimizer/cluster-optimizer     0.1.0           1.1.0           A Helm chart for cluster optimizer

    It is known that the repository contains a chart named cluster-optimizer/cluster-optimizer. 可知，仓库中包括一个名为cluster-optimizer/cluster-optimizer的Chart。

  4. To install the demo chart, use the following command: 通过如下命令安装demo Chart：

        helm install demo cluster-optimizer/cluster-optimizer -n wiseinf-system --create-namespace

    Here, -n wiseinf-system indicates installing demo under the namespace wiseinf-system, and the –create-namespace parameter indicates creating this namespace. The output is as follows: 其中，-n wiseinf-system表示在名称空间wiseinf-system下安装demo， 参数--create-namespace表示创建此名称空间，输出如下：

    NAME: demo
LAST DEPLOYED: Thu Mar 28 15:54:42 2024
NAMESPACE: default
STATUS: deployed
REVISION: 1
...

    Helm has successfully deployed the demo chart. 说明Helm已经成功部署demo Chart。

  5. Check if optimizer is running using following command: 通过如下命令检查optimizer是否正常运行：

        kubectl get pods -n wiseinf-system

    The output should look like this: 输出如下：

    NAME                                                  READY   STATUS    RESTARTS   AGE
cluster-optimizer-demo-7bfdfdf954-2cgqb               2/2     Running   0          108s
cluster-optimizer-demo-optimizerdb-5d6cdb6b9f-94lc7   1/1     Running   0          108s

    From the output, it can be seen that Helm deployed two pods. Among them, cluster-optimizer-demo-7bfdfdf954-2cgqb is the pod of Cluster Optimizer, and cluster-optimizer-demo-optimizerdb-5d6cdb6b9f-94lc7 is its dependent MySQL database. 从输出可以看到，Helm部署了两个Pod，其中cluster-optimizer-demo-7bfdfdf954-2cgqb是 Cluster Optimizer的Pod， cluster-optimizer-demo-optimizerdb-5d6cdb6b9f-94lc7是其依赖的Mysql数据库。

    Note: If you intend to use a deployed database, please refer to Using a Deployed Database.

  6. reate a port-forward to access optimizer service: 通过如下命令创建 port-forward 来访问 optimizer 服务：

        kubectl port-forward svc/optimizer 8000:80  -n wiseinf-system

  7. Open your browser and visit url http://localhost:8000/.

  8. If you can see the home page of Cluster Optimizer, you have successfully setup optimizer.

  9. To delete the demo chart, using following command:通过命令可以删除demo Chart：

    helm uninstall demo

Using a Deployed Database

If you want to use a deployed database or want to deploy the database by yourself, please follow the instructions below:

We assume the installation script library has been cloned into setup subdirectory of user’s home directory.

  1. Clone installation script into local using git:

        cd ~
    git clone https://github.com/wiseinf/setup.git

  2. Login into mysql database using:

        mysql -h127.0.0.1 -uroot

    The 127.0.0.1 is the host address of the MySQL database, which you need to replace with the actual address you are using.

    Note: Please make sure the database is accessible from the kubernetes cluster.

  3. Create database schema optconf(used to store configurations) and optdata(used to store data)

        mysql> source ~/setup/cluster-optimizer/sql/create_database.sql

    Note: Please be aware of the risk of data loss. The script will drop schema optconf and optdata first and then create new schemas. If you already have an optconf or an optdata schema, your data in this schema will be lost.

  4. Create a new user and grant permission to the new user. The user name is optimizer, password is Optimizer~, you can change it to your own.

        mysql> source ~/setup/cluster-optimizer/sql/create_user.sql

  5. Create a file values.yaml and edit its content as follows:

    optimizedb:
  conf: 
    dbHost: 127.0.0.1
    schema: optconf
    port: "3306"
    user: optimizer
    password: Optimizer~
  data:
    dbHost: 127.0.0.1
    schema: optdata
    port: "3306"
    user: optimizer
    password: Optimizer~

    Here, optimizedb.conf provides configuration information for the optconf schema, while optimizedb.data provides configuration information for the optdata schema. In which, dbHost is the database address, schema is the database schema, port is the port, and user and password are the username and password respectively.

  6. Install the demo chart using the following command:

        helm install demo cluster-optimizer/cluster-optimizer -f values.yaml -n wiseinf-system

    You can also adjust the database settings by using helm upgrade command as follows:

        helm upgrade demo cluster-optimizer/cluster-optimizer -f values.yaml --namespace wiseinf-system

  7. Restart the application and check the log output of the Cluster Optimizer to verify if the database configuration information matches your expectations. For example:

    {
    ...
    "optconf": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optconf?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    "optdata": {
        "dataSource": {
            "name": "optimizer:Optimizer~@tcp(127.0.0.1:3306)/optdata?createDatabaseIfNotExist=true\u0026charset=utf8mb4\u0026parseTime=true\u0026loc=Local",
            "maxOpenConns": 10
        }
    },
    ...
}

    At the same time, also check for any database-related errors. If there are no relevant errors, it indicates that the new database has been successfully used.
