Ruyi: Cloud native cluster autoscaling engine

Ruyi (Ruyi, meaning that clusters and nodes can be expanded and reduction as desired) is a cloud native cluster autoscaling engine. It provides both autoscaling and virtual capacity. With these capabilities, it can help you save up to 50% of cloud native IT costs.

 

Documentation Request a trial

Cluster autoscaling

Pod request based autoscaling: Significantly reduces the risk of service unavailability caused by many autocaling mechanism based on pod pending.

Elegant scaling strategy effectively reduces the risk of service unavailability caused by deleting nodes.

Seamless integration with the cluster autoscaling mechanism provided by cloud service providers.

Virtual capacity

Node capacity can increased by 3 times (or even more) of its original capacity under low resource utilization, and support scheduling more pods (workloads) on the node.

Especially suitable for development, testing environment with low resource utilization.

Multi-cloud support

Supports AWS and Alibaba Cloud.
